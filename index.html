PORTFOLIO

The Sandbox style metaverse - 2D world map

I took an existing world map asset from the Unity Asset Store, moulded it to fit design specifications and incorporated it into our project. Modifications include: profiling and fixing performance and memory leaks, grid and map style changes, region detection and generating grid data JSON files

Pokémon GO style pet collector - 3D GPS map

I collaberated on this feature with our team and focused on fixing bugs. One issue I was able to isolate and fix was a platform specific tile loading bug. Another Android specific issue required me to dive into some Kotlin code and fix an implementation of an Android service.

Merge style game

Dragging, dropping, merging, fun. One of the things I've done a bunch is implementing sounds, particle effects and DOTweens (a unity animation engine).

'AddComponent' tracking and editor script

The performance of 'AddComponent' is quite bad so we were looking for a way to avoid instances of this. I tracked wherever this happened at runtime, storing relevant heirarchy information in JSON files. I created an editor script to run through our prefabs and add the tracked components upfront.

Leaderboards

Abstraction, scrolls with resuable elements, pagination, request and response, all very usual for me. I didn't make the backend for these leaderboards but I've debugged and fixed a few bugs on the server-side code. These leaderboards are using an Amazon DynamoDB.

Weekly Riddle

I've done a few client-side GraphQL implementations and this is one of them. I'm very comfortable with the Unity UI system and have done a lot myself in that regard. Those rewards at the bottom are tracked on the the client and server using Entities from the Entity Component System, Entitas.
